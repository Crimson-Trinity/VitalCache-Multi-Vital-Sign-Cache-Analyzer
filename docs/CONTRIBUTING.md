# Contributing to VitalCache

Thank you for considering contributing to VitalCache! This document provides guidelines for contributing to the project.

## How to Contribute

1. **Fork** the repository
2. **Create** a feature branch (`git checkout -b feature/your-feature`)
3. **Commit** your changes (`git commit -m 'Add: your feature description'`)
4. **Push** to the branch (`git push origin feature/your-feature`)
5. **Open** a Pull Request with a clear description

## Code Style

- Use clear, descriptive variable names
- Comment all functions with `@brief`, `@param`, and `@return` tags
- Keep functions focused and single-purpose
- Use `#define` for configurable constants

## Reporting Issues

- Use the GitHub Issues tab
- Include your hardware setup (ESP32 board version, sensor models)
- Include Serial Monitor output showing the issue
- Describe steps to reproduce

## Areas for Contribution

- Additional cache mapping techniques (e.g., N-way set associative)
- New replacement policies (LRU, LFU, Random)
- Improved sensor signal processing
- Web/mobile dashboard integration
- Documentation improvements
- Unit tests and simulation scripts
